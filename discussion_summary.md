# 自動車車室内UX評価 — 検討の視点と方向性

## 1. 検討の背景と前提条件

### 1.1 関心のある開発フェーズ

| フェーズ | 評価回数（想定） | 主な目的 |
|---|---|---|
| コンセプト初期 | 3案比較 × 2まわし | 方向性の絞り込み・案の精緻化 |
| プロトタイプ検証 | 2回 | 実装品の問題発見・改善確認 |
| バリデーション | 1回 | 合否判定＋提案素材作成 |
| 社内・顧客提案 | — | 意思決定支援・アピール |

### 1.2 評価対象のUX側面

- **HMI/UIインタラクション** — 操作性、視認性、タスク成功率、認知負荷
- **感性価値** — 製品を見たり触ったりした際のユーザーの感じ方を計測・意味づけ・スコア化したい（具体的な方法論は未確立）

### 1.3 評価の主体

- **短期**：実ユーザー（被験者）による評価が主体
- **将来**：仮想ユーザー（Synthetic User）やAIエージェントとの協働へ進化させたい
- **目標**：2030年以降を含む長期ビジョンまで描く（短期はより具体的に）

### 1.4 既存資産と組織的目的

- **既存設備**：実車プロトタイプ（生体計測・骨格検知の測定系設置済み）
- **HMI切替方式**：実製品の物理的な設置・付替え＋スクリーンにHMI画像表示による素早い切替
- **組織目的**：
  - 車室内UX評価プロセスの新規構築・体系化（A）
  - 既存設備とデジタル技術の統合（B）

### 1.5 評価結果の使い方

- **企画・設計へのフィードバック**（改善点の特定と提示）
- **顧客へのアピール材料**（価値訴求のエビデンス）

### 1.6 評価の信頼性方針

- **段階的アプローチ**を採用：初期は粗く速く、最終段階で精密に

### 2.2 評価精度に関する学術的知見

- **芝浦工業大学**：VR空間のUX評価は実空間と概ね整合する（特に使いやすさ・動線）。ただしVR酔い・身体動作制限による差異あり。VRは「実空間UX評価のスクリーニング」として有効
- **DriverDoubles（Aalto大学）**：Synthetic Userは構造化タスク（判断型）で高い再現性。感情・ブランド体験の評価では不十分。「実ユーザーの代替ではなく補完」

### 2.3 感性評価に必要なデータ（文書群から）

Synthetic Userの学習データ文書より、感性評価に関連する計測データ：
- 視線データ（Eye tracking、視線ヒートマップ）
- 生体データ（心拍、HRV、呼吸、皮膚電気反応）
- 行動データ（接触動作、操作時間、探索パターン）
- 主観データ（SD法、インタビュー、アンケート）

## 3. 持つべき6つの視点

### 視点1：評価対象の二層構造 — 「機能UX」と「感性UX」を分けて設計する

| 層 | 内容 | 計測しやすさ | 代表手法 |
|---|---|---|---|
| **機能UX層** | HMI/UIの操作性、視認性、タスク成功率、操作時間、エラー率 | ◎ 構造化しやすい | VR-Buck、Varjo XR-3、Unified HMI |
| **感性UX層** | 「上質に感じる」「心地よい」「ブランドらしい」といった主観的価値 | △ 直接計測困難 | CAVE面質評価、生体×主観統合 |

**感性スコア化の3段階モデル：**

```
[生体信号・行動データ]  →  [心理状態の推定]  →  [感性価値スコア]
       計測層                   解釈層               評価層
```

- **計測層**：HRV、EDA/GSR、視線分布・停留時間、表情筋電位(EMG)、骨格姿勢変化
- **解釈層**：生体データから「快/不快」「緊張/弛緩」「注意集中/散漫」等の心理次元へマッピング（Russell's Circumplex Model等を基盤）
- **評価層**：心理次元の複合スコアを設計意図（「上質」「直感的」「先進的」等）と紐付け、感性プロファイルを生成

### 視点2：段階的精度設計 — 各フェーズで「何をどの精度で測るか」を定義する

| フェーズ | 目的 | 機能UX評価 | 感性UX評価 | 主な手法 |
|---|---|---|---|---|
| コンセプト1回目 | 方向性の絞り込み（3→2案） | スクリーン表示HMI操作の相対比較（タスク完了率・操作時間） | 第一印象評価（SD法）＋視線ヒートマップ | スクリーン投影＋簡易操作＋主観アンケート |
| コンセプト2回目 | 案の精緻化（2案深掘り） | 同上＋エラー分析 | 生体計測追加（HRV、EDA）で無意識反応を捕捉 | 実車プロト内スクリーン＋生体計測 |
| プロト1回目 | 実装品の問題点発見 | 実製品での操作性検証、認知負荷評価 | 生体＋骨格で操作中の身体反応を統合分析 | 実製品設置＋フル計測系 |
| プロト2回目 | 改善確認＋最終判断材料 | 改善前後の定量比較 | 感性スコアの変化検証 | 同上＋比較分析 |
| 最終バリデーション | 合否判定＋提案素材作成 | 全指標の閾値確認 | 感性プロファイルの完成 | フルセット＋映像記録 |
| 社内・顧客提案 | 意思決定支援・アピール | — | — | 可視化ダッシュボード＋体験デモ |

### 視点3：物理×デジタルのハイブリッド評価設計

既存の実車プロトタイプ（生体計測・骨格検知付き）にデジタル技術を重ねる構成：

```
┌─────────────────────────────────────────────────┐
│           実車プロトタイプ（物理層）              │
│  ┌──────────┐  ┌──────────┐  ┌──────────────┐   │
│  │実製品設置 │  │スクリーン│  │生体計測系    │   │
│  │(付替可能) │  │(HMI表示) │  │骨格検知系    │   │
│  └──────────┘  └──────────┘  └──────────────┘   │
├─────────────────────────────────────────────────┤
│           デジタル拡張層（統合すべき領域）        │
│  ┌──────────┐  ┌──────────┐  ┌──────────────┐   │
│  │MRヘッド  │  │視線追跡  │  │データ統合    │   │
│  │セット    │  │(Varjo等) │  │分析基盤     │   │
│  │(AR重畳)  │  │          │  │              │   │
│  └──────────┘  └──────────┘  └──────────────┘   │
└─────────────────────────────────────────────────┘
```

**統合の優先順位（短期）：**

1. **データ統合分析基盤**の構築 — 生体計測・骨格・視線・操作ログを時間同期して統合分析できる環境（産総研Xperigrapher参考）
2. **スクリーンHMI表示のソフト管理** — HMI画面のソフトウェア瞬時切替（SOAFEE/Unified HMI型アーキテクチャ参考）
3. **MRヘッドセットの導入検討** — 高精細MR（Varjo XR-3等）で実車内装に仮想HMI表示を重畳

### 視点4：評価出力の二方向設計 — 「改善FB」と「価値訴求」

| 出力方向 | 必要な情報 | 形式 |
|---|---|---|
| 企画・設計へのフィードバック | どこが・なぜ・どの程度問題か。改善の方向性 | 診断レポート：ヒートマップ、操作フロー上の問題箇所マーク、感性スコアのレーダーチャート、生体反応タイムライン |
| 顧客への価値訴求 | 「このHMIはここが優れている」エビデンス | プレゼン素材：比較スコア可視化、体験映像＋生体反応オーバーレイ、感性プロファイルの言語化 |

### 視点5：評価主体の進化ロードマップ — 実ユーザーからAI協働へ

```
短期(〜2027)          中期(2028〜2030)           長期(2030〜)
─────────────────────────────────────────────────────────
[実ユーザー主体]  →  [AI補完型]            →  [AI協働型]

・実車プロト内で      ・Synthetic Userで        ・多エージェント
 被験者がタスク実行    コンセプト段階の          パイプラインで
・生体計測＋主観評価    スクリーニング自動化      自動評価ループ
・手動分析             (DriverDoubles型)        (N-iX型)
                     ・AI専門家エージェント    ・実ユーザーは
                      による事前レビュー        感性検証のみに
                      (多視点自動評価)          集中
                     ・生体データの             ・感性AI
                      AI自動解釈               (生体→感性の
                                               自動推定モデル)
```

**注意事項（DriverDoubles知見より）：**
- 構造化タスク（「通知に気づくか」「ボタンを押せるか」）→ Synthetic Userで代替しやすい
- 感性評価（「上質に感じるか」「心地よいか」）→ 当面は実ユーザーが不可欠
- AI化が先行するのは機能UX層。感性UX層は実ユーザーデータ蓄積→AIモデル学習の順序

**中期への布石**：実ユーザー評価データを将来のAIモデル学習に使えるフォーマットで蓄積すること。特に「生体データ×主観評価×HMI仕様」の三つ組データセットが感性AIの訓練データになる。

### 視点6：感性評価の方法論的基盤

#### A. マルチモーダル計測の三角測量

| データ種 | 取得方法（既存設備で可能） | 推定できる心理次元 |
|---|---|---|
| 生理指標 | HRV、EDA、呼吸、表情筋 | 覚醒度、快/不快、緊張/弛緩 |
| 行動指標 | 視線（停留・サッケード）、骨格姿勢、接触動作 | 注意配分、探索行動、接近/回避 |
| 言語指標 | 発話プロトコル、SD法、自由記述 | 概念的評価、意味づけ、ブランド印象 |

#### B. 感性ワードの構造化（SD法の拡張）

「上質」「先進的」「安心感」等の感性ワードを事前定義し、各ワードに対応する生体・行動パターンの仮説を立てて検証する：

```
感性ワード「上質」
  └→ 仮説：視線が滑らかに流れる（急速サッケード少）
  └→ 仮説：接触時間が長い（回避しない）
  └→ 仮説：HRV高周波成分増加（リラックス）
  └→ 仮説：SD法「洗練された-粗野な」スコアと相関
```

この仮説→検証のループを回すこと自体が、短期の最重要アクションとなる。
