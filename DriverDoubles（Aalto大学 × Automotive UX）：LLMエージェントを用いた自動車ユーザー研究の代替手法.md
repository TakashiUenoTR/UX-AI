
# DriverDoubles（Aalto大学 × Automotive UX）：LLMエージェントを用いた自動車ユーザー研究の代替手法 詳細整理
（Aalto大学 2025年修士論文・研究発表に基づくまとめ）

## 1. 概要
**DriverDoubles** は、Aalto大学（フィンランド）が 2025 年に発表した研究で、
**Generative Agents（LLMエージェント）を用いて自動車ユーザーの行動をシミュレートし、実ユーザー研究を部分的に代替する**ことを目的としている。citeturn30search237

従来の自動車UX研究（運転中の行動観察・割り込み許容度評価・感情推定など）は、
- コストが高い
- 大規模データ収集が困難
- 安全制約により危険シナリオの再現が難しい
という課題があった。

DriverDoubles は、これらを **LLMベースの Synthetic User（仮想ユーザー）** によって補完し、
UX評価を高速化する “次世代のユーザー研究手法” として注目されている。citeturn30search237

---

## 2. DriverDoubles の研究目的
DriverDoubles の中心的な目的は次の3点：

### ● ① UX研究の初期段階を高速化
リアルなユーザー試験を行う前に、AIエージェントを使ってデザイン案の方向性を素早く検証する。

### ● ② 危険シナリオや特殊条件の再現
実車で行えないシナリオ（危険運転、極端な集中状態、疲労など）をAIで安全に再現する。

### ● ③ 自動車における“人間状態”の理解
割り込み許容度（Interruptibility）、感情状態（Emotion Recognition）など、
**ドライバーの認知的・心理的状態を近似**する Synthetic User を構築すること。

研究は、以下の 2 つの実車データセットを基準として比較評価された。citeturn30search237
- **Interruptibility（割り込み許容度）評価タスク**
- **Emotion Recognition（感情推定）タスク**

---

## 3. DriverDoubles の技術構造（Generative Agents Architecture）
研究では、LLMベースの Synthetic User を次の構成で実現している：citeturn30search237

### ● 1. Synthetic Persona（人格・設定）
- 年齢、運転熟練度、注意力、性格特性などを付与
- エージェントはこの人格に基づき行動判断を行う

### ● 2. Driving Context（運転コンテキスト）
- 周囲交通、速度、車間距離、認知負荷、UIへの注視状態など
- 多モーダルな運転情報をテキスト化して入力

### ● 3. Goal Conditioning（目的条件）
例：
-「この状況でスマホ通知に反応するか？」
-「この運転映像を見てどんな感情になるか？」

### ● 4. Reasoning Engine（LLM推論）
- GPT系モデルを想定したテキスト推論
- persona × context × goal に基づき行動・回答を生成

### ● 5. Evaluation Pipeline（人間データとの比較）
- 多数の Synthetic User を生成し、実験参加者データと比較
- 一致率・相関・機械学習モデルによる再現性を検証

---

## 4. 評価実験：Synthetic User は本当に人間を模倣できるか？
DriverDoubles は、2つのUXタスクで Synthetic User の再現性を検証した。citeturn30search237

## ● 4-1. タスク①：割り込み許容度（Interruptibility）
### ▶ 目的
運転中に通知（例：車載UI、スマホ）が来た際、
「反応するべきか？」「無視するべきか？」を判断するモデル。

### ▶ 結果
- **人間データとの一致率が高かった**（群集投票による多数決で整合）
- Synthetic User が示す判断傾向は、人間参加者のパターンと類似
- Structured Task に強く、再現性が高い

→ **結論：UX初期検証やHMI評価で有効**

---

## ● 4-2. タスク②：感情推定（Emotion Recognition）
### ▶ 目的
運転映像を見て、ドライバーがどんな感情になるかAIに推定させるタスク。

### ▶ 結果
- **人間データとの一致率は低い**
- 感情推定は暗黙的で曖昧な判断が多く、LLMの限界が顕著

→ **非構造タスク（感情・印象評価）ではまだ不十分**

---

## 5. 研究が示す知見（メリットと限界）
### ✔ Synthetic User のメリット
- UX初期段階の高速な方向性検証が可能
- 危険な運転シナリオを安全に再現できる
- 大規模・反復テストが低コストで実行可能
- 実ユーザーとの一致が高いタスクも存在（Interruptibility）

### ✔ Synthetic User の限界
- 情動・内的感情の推定は不得意
- コンテキスト入力の品質に強く依存
- エージェントのパーソナ設計によってバイアスが生じる
- 最終検証は必ず“実ユーザー”が必要、と研究も明言citeturn30search237

---

## 6. DriverDoubles の位置づけと応用可能性
### ● 自動車UXのどの領域で有効？
#### ① HMI（操作性）検証
- ボタン配置
- 車載通知の煩わしさ
- 認知負荷の評価

#### ② ADAS（運転支援）シナリオ評価
- ドライバーの注意散漫状態の再現
- 危険シナリオでの反応シミュレーション

#### ③ UX初期コンセプト検証
- 新機能の案が “受け入れられそうか” をSynthetic Userで短時間に確認

### ● OEMやサプライヤーが導入した場合の効果
- 試作車が無くても初期段階のUX検証が可能
- 国地域ごとに“仮想ドライバーモデル”を作成可能（文化差のテスト）
- 実験コスト・時間の大幅削減

---

## 7. まとめ
DriverDoubles は、
> **LLMエージェントが“運転中の人間行動”をどこまで代替できるかを体系的に検証した世界初の研究の一つ**。

研究の結論：
- **構造化されたタスク（割り込み判断など）は高い再現性**
- **感情推定などの非構造タスクは不十分**
- **Synthetic User は“実ユーザーの代替ではなく補完”として有効**

特に、自動車メーカーの：
- HMIデザイン初期の方向性検証
- ADAS UX評価の危険シナリオ再現
- 大規模・反復シミュレーション
に大きな可能性を持つ。

今後、Generative Agents × Driving Simulator × マルチモーダルAI が統合されることで、
**“デジタル上に仮想ユーザー群を作り、実験室を拡張する”** 新しいUX研究手法が主流になると考えられる。
