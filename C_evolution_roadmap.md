# C. 進化ロードマップ — 実ユーザー評価からAI協働評価へ

## 1. 本書の目的

自動車車室内UX評価を、現在の実ユーザー主体の評価から、Synthetic User・AIエージェントとの協働による評価へ段階的に進化させるための具体施策を、短期・中期・長期の時間軸で描く。

## 4. 中期（2028〜2030）：AI補完型への移行

### Phase 3：分析のAI化と効率化（2028年）

#### 施策一覧

| # | 施策 | 具体内容 | 技術要素 |
|---|---|---|---|
| 3-1 | 生体データ自動前処理 | HRV、EDA、視線データの前処理（ノイズ除去、特徴量抽出）を自動化するパイプライン構築 | Python/MATLAB + 信号処理ライブラリ |
| 3-2 | 心理次元自動算出 | 生体・行動データから4心理次元（Valence, Arousal, Approach, Fluency）を自動計算するモジュール構築 | 機械学習（Random Forest等）+ 蓄積データ |
| 3-3 | 感性スコア自動算出 | 心理次元→感性ワードスコアの自動変換＋プロファイルカード自動生成 | 回帰モデル + テンプレートエンジン |
| 3-4 | リアルタイムダッシュボード | 評価セッション中にリアルタイムで生体反応・視線データを可視化。実験者が「今何が起きているか」を把握 | Web UI + ストリーミングデータ処理 |
| 3-5 | 比較分析の半自動化 | 案A vs 案B比較、改善前後比較のレポートを半自動生成 | テンプレート＋統計処理自動化 |

#### この段階で変わること
- **評価にかかる分析工数が大幅に削減**（手動分析→自動算出＋人間確認）
- **評価セッション中のリアルタイムフィードバック**が可能に
- データ品質の自動チェックにより、異常データの即時検出が可能

#### 評価プロセスの変化（Phase 4時点）

```
          【従来（Phase 2）】                 【Phase 4以降】
                                        
コンセプト: 3案 → 実ユーザー評価      3案 → Synthetic Userスクリーニング
           → 2案絞り込み                    → AI専門家レビュー
                                             → 2案に絞り込み
                                             → 実ユーザー深掘り評価

プロト:    実製品 → 実ユーザー評価      実製品 → AI自動品質チェック
           → 問題発見                        → 実ユーザー感性評価
                                             → AI感性予測との比較検証

バリデ:    実ユーザー → 手動分析         実ユーザー → AI自動分析
           → レポート                        → 自動レポート生成
                                             → 人間確認・編集
```

#### Synthetic User導入時の適用範囲と制約

DriverDoubles研究の知見に基づく適用可否：

| 評価タスク | Synthetic User適用 | 根拠 |
|---|---|---|
| UI要素の発見可能性（ボタンを見つけられるか） | ◎ 適用可 | 構造化タスク、高再現性 |
| 操作フローの論理性（手順が理解できるか） | ◎ 適用可 | 構造化タスク、高再現性 |
| 通知の割り込み許容度 | ◎ 適用可 | DriverDoublesで高一致率を確認 |
| 情報の可読性・視認性 | ○ 部分的に適用可 | ルールベースで判定可能な部分あり |
| 操作の「気持ちよさ」 | △ 限定的 | 感性要素が大きく、LLMでは困難 |
| 素材・質感の印象 | × 適用不可 | 物理的感覚の再現が不可能 |
| ブランド体験の一貫性 | × 適用不可 | 文化・経験依存が大きい |
| 感情的反応 | × 適用不可 | DriverDoublesで低一致率を確認 |

## 6. 短期→中期への橋渡し：今から準備すべきこと

短期（実ユーザー主体）から中期（AI補完型）への移行を円滑にするため、**短期の段階で意図的に埋め込んでおくべき布石**：

### 6.1 データ蓄積の設計

| 準備項目 | 目的 | 具体的な対応 |
|---|---|---|
| HMI仕様の構造化記録 | 将来の「HMI仕様→感性スコア」予測モデルの入力データ | 評価対象HMIの仕様を統一フォーマットで記録（画面レイアウト、色、フォント、物理スイッチ仕様等） |
| 生体データの生データ保存 | 将来的な再分析・新手法適用に備える | 前処理済みデータだけでなく、RAWデータも保存 |
| 主観評価の自由記述保存 | 将来のNLP分析やLLMファインチューニング素材 | インタビュー音声の文字起こし、自由記述のテキスト保存 |
| 被験者プロファイル | Synthetic Userのペルソナモデル構築素材 | 年齢、運転経験、ITリテラシー、嗜好等を構造化記録 |

### 6.2 評価フレームのモジュール化

将来のSynthetic User・AIエージェント導入時にプロセスを組み替えやすくするため、評価の各ステップを「モジュール」として独立性高く設計する：

## 7. リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| 生体-感性マッピングの精度が上がらない | 感性スコアの信頼性に疑問 → 社内説得困難 | 主観評価との併用を維持。スコアは「参考値」として扱い、主観データとの乖離パターンを記録して改善 |
| Synthetic Userの感性評価精度が実用レベルに達しない | 感性領域でのAI化が遅延 | 機能UX領域でのAI化を先行し、感性は実ユーザー評価を維持。感性AIは研究テーマとして継続 |
| 被験者確保が困難 | 評価回数・サンプルサイズが不足 | 社内被験者プールの早期構築。協力企業・大学との被験者共有ネットワーク |
| 計測機器の陳腐化 | 新手法への対応困難 | データフォーマットを機器非依存に設計。分析パイプラインの入力インターフェースを抽象化 |
| 組織内の理解不足 | 予算・リソース確保困難 | 早期の成功事例作成（Phase 2-3の顧客提案実績）で価値を実証。感性プロファイルカードの視覚的インパクトを活用 |

## 8. 投資対効果の見通し

### 8.1 短期（Phase 1-2）で得られる価値

- **評価プロセスの体系化**による意思決定の高速化・属人性排除
- **感性の定量的言語化**による社内コミュニケーション改善（「なんとなく良い」→「上質さスコア78」）
- **顧客提案力の強化**（エビデンスベースの訴求が可能に）
- **設計手戻りの削減**（コンセプト段階での早期問題発見）

### 8.2 中期（Phase 3-4）で得られる価値

- **分析工数の大幅削減**（手動数日→自動数時間）
- **評価カバレッジの拡大**（Synthetic Userで100パターンのスクリーニング）
- **開発リードタイム短縮**（AI事前スクリーニング→実ユーザー評価の絞り込み）

### 8.3 長期（Phase 5-6）で得られる価値

- **設計変更→評価→フィードバックの即時ループ**（日単位→時間単位）
- **感性予測による設計最適化**（試作前に感性スコアを予測）
- **グローバル展開時のペルソナ対応力**（国・地域別の仮想ユーザーモデル）
- **SDV時代のOTAアップデート対応**（更新ごとの自動品質確認）

## 9. まとめ：進化の本質

```
                     実ユーザー評価
                     の「蓄積データ」
                          │
                          ↓
         ┌────────────────────────────────┐
         │  データが資産になる            │
         │  （生体×主観×HMI仕様の三つ組）│
         └────────────────┬───────────────┘
                          │
              ┌───────────┴───────────┐
              ↓                       ↓
      ┌───────────────┐      ┌───────────────┐
      │ AIの学習材料  │      │ 評価基準の    │
      │ になる        │      │ 精緻化になる  │
      └───────┬───────┘      └───────┬───────┘
              │                       │
              ↓                       ↓
      ┌───────────────┐      ┌───────────────┐
      │ Synthetic User│      │ 品質閾値の    │
      │ 感性AI        │      │ エビデンス化  │
      └───────────────┘      └───────────────┘
```

**進化の核心**：短期の実ユーザー評価は「目の前の製品を良くする」だけでなく、**将来のAI協働評価を実現するための訓練データを生み出している**。この認識を持って短期のデータ蓄積設計を行うことが、長期ビジョン実現の最大の鍵である。
