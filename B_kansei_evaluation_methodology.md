# B. 感性評価の方法論 — 計測プロトコル設計

## 1. 本書の目的

自動車車室内の感性価値（ユーザーが製品を見たり触ったりした際に「どう感じるか」）を、既存の生体計測・骨格検知設備を活用して**計測→解釈→スコア化**するための方法論と具体的プロトコルを定義する。

---

## 2. 感性評価の基本構造

### 2.1 三層パイプライン

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   計測層     │     │   解釈層     │     │   評価層     │
│              │ ──→ │              │ ──→ │              │
│ 生体・行動・ │     │ 心理次元への │     │ 感性ワード   │
│ 言語データ   │     │ マッピング   │     │ スコア化     │
│ の取得       │     │              │     │              │
└──────────────┘     └──────────────┘     └──────────────┘
  What（何が         Why（なぜそう        So What（だから
  起きているか）     なっているか）       どうなのか）
```

### 2.2 なぜ三層が必要か

- **計測層だけ**では「HRVが下がった」という事実しか分からない
- **解釈層**を経て「緊張が高まった」という心理状態に変換される
- **評価層**で初めて「"上質さ"を感じていない可能性がある」という設計判断に繋がる

この変換過程を明示的に設計しないと、計測データがあっても意思決定に使えない。

## 3. 計測層：何をどう測るか

### 3.1 計測モダリティの構成

| モダリティ | 計測項目 | センサー/手法 | サンプリング | 推定可能な情報 |
|---|---|---|---|---|
| **生理** | 心拍変動（HRV） | 胸部ベルト or 手首センサー | 連続（1000Hz推奨） | 自律神経バランス、ストレス、リラックス |
| 生理 | 皮膚電気反応（EDA/GSR） | 指先 or 手のひら電極 | 連続（10Hz以上） | 情動的覚醒、驚き、興味 |
| 生理 | 呼吸 | 胸部ベルト or 鼻腔センサー | 連続 | 覚醒度、リラックス度 |
| 生理 | 表情（可能であれば） | カメラ＋表情認識AI | 30fps以上 | 快/不快、驚き、困惑 |
| **行動** | 視線 | アイトラッカー | 60Hz以上 | 注意配分、興味対象、探索パターン |
| 行動 | 上半身姿勢 | 骨格検知（深度カメラ等） | 30fps以上 | 接近/回避、前傾/後傾、リーチ動作 |
| 行動 | 手の動き | 骨格検知 | 30fps以上 | 接触行動、操作軌跡、ためらい |
| 行動 | 接触時間・回数 | 操作ログ＋映像 | イベント単位 | 興味の持続、探索意欲 |
| **言語** | SD法 | 紙 or タブレット | タスク後 | 概念的感性評価 |
| 言語 | 発話プロトコル | 音声録音 | 連続 | リアルタイムの感想・気づき |
| 言語 | 半構造化インタビュー | 対面 or録音 | セッション後 | 深層的な意味づけ |
| 言語 | 自由記述 | タブレット | セッション後 | 非誘導の印象記録 |

### 3.2 計測の時間構造

一つの評価セッションにおける計測のタイムライン：

```
時間 →
├─ 安静ベースライン(3min) ─┤─ 第一印象(2min) ─┤─ 操作タスク(10-15min) ─┤─ 自由探索(5min) ─┤─ 安静回復(2min) ─┤─ 主観評価(10min) ─┤

[生体計測：全区間連続]
[視線追跡：第一印象〜自由探索]
[骨格検知：操作タスク〜自由探索]
[映像記録：全区間]
[発話プロトコル：操作タスク〜自由探索]
[SD法・インタビュー：主観評価区間]
```

#### 各区間の目的

| 区間 | 時間 | 目的 | 取得データ |
|---|---|---|---|
| **安静ベースライン** | 3分 | 個人差の正規化基準を取得 | HRV・EDA・呼吸の安静時水準 |
| **第一印象** | 2分 | 初見の感性反応を捕捉（操作前） | 視線走査パターン、EDAの初期反応、第一印象SD法（簡易版） |
| **操作タスク** | 10〜15分 | 機能UXと操作中の感性反応 | 全計測系フル稼働 |
| **自由探索** | 5分 | 非タスク状態での自発的興味行動 | 視線・手の動き（何を自発的に見る・触るか） |
| **安静回復** | 2分 | 体験後の生理的回復パターン | HRV・EDA回復速度（感情残余の指標） |
| **主観評価** | 10分 | 言語化された感性評価 | SD法＋インタビュー＋自由記述 |

### 3.3 第一印象計測の詳細

感性評価において「最初の2分」は特に重要。言語的思考が介入する前の無意識反応を捕捉する。

**プロトコル：**
1. 被験者を目を閉じた状態で着座させる
2. 「目を開けてください。自由に周囲を見てください」と教示
3. 2分間、一切の操作指示を出さない
4. 2分後、簡易SD法（5対程度の形容詞対）を即座に回答させる

**計測ポイント：**
- 視線の最初の走査パス（何を最初に見るか、どの順序で見るか）
- EDAの初期ピーク（最初の10〜30秒にピークが出るか）
- 表情変化（微笑み、眉の動き等）

## 4. 解釈層：生体・行動データから心理次元への変換

### 4.1 心理次元の定義

Russell's Circumplex Model を基盤にしつつ、車室内UX評価に特化した4次元モデルを採用する：

```
          覚醒（高）
            ↑
            │
   興奮     │     緊張
  ワクワク   │    ストレス
            │
  ──────────┼──────────→ 快（右）
  不快（左） │
            │
   退屈     │     安心
   無関心   │    リラックス
            │
            ↓
          覚醒（低）
```

**4つの心理次元：**

| 次元 | 定義 | 感性評価における意味 |
|---|---|---|
| **Valence（快/不快）** | ポジティブ/ネガティブの方向 | 基本的な好み・受容 |
| **Arousal（覚醒度）** | 活性化/沈静化の程度 | 興味・注意のレベル |
| **Approach/Avoidance（接近/回避）** | 対象に近づく/遠ざかる傾向 | 製品への積極的関与 |
| **Fluency（処理流暢性）** | 情報処理のスムーズさ | 直感的理解度・違和感の有無 |

### 4.2 生体・行動指標から心理次元へのマッピング

| 心理次元 | 主要指標 | 補助指標 | 変換ロジック |
|---|---|---|---|
| **Valence（快/不快）** | EDAトニックレベル＋表情（笑顔/しかめ面） | HRV-HF成分、自由探索時の接触回数 | EDA上昇×表情ポジティブ→快 / EDA上昇×表情ネガティブ→不快 |
| **Arousal（覚醒度）** | EDAフェイジック反応（ピーク振幅・頻度） | 瞳孔径変動、呼吸数変化 | ベースラインからの変化率で算出 |
| **Approach/Avoidance** | 上半身前傾角度、手の到達距離 | 視線滞留時間、接触時間 | 前傾＋長時間注視→接近 / 後傾＋短時間注視→回避 |
| **Fluency（処理流暢性）** | 視線サッケード頻度、操作ためらい時間 | HRV-LF/HF比（認知負荷指標） | サッケード少＋ためらい少→流暢 / サッケード多＋ためらい多→非流暢 |

### 4.3 マッピングの校正プロセス

初期段階では上記マッピングは「仮説」に過ぎない。以下のプロセスで校正する：

```
               ┌──────────────────┐
               │ 仮説マッピング   │
               │ の設定           │
               └────────┬─────────┘
                        ↓
               ┌──────────────────┐
               │ パイロット評価   │──→ 生体データ取得
               │ (N=5〜10)       │──→ 主観評価取得
               └────────┬─────────┘
                        ↓
               ┌──────────────────┐
               │ 相関分析         │
               │ 生体指標 ←→     │
               │ 主観評価         │
               └────────┬─────────┘
                        ↓
                 相関が高い    相関が低い
                    ↓              ↓
               マッピング      マッピング
               を維持          を修正・再検証
```

**校正のポイント：**
- 最初は主観評価（SD法）を「正解」として、生体指標がそれを予測できるかを検証
- 相関が取れた指標の組み合わせを「信頼できるチャネル」として採用
- プロジェクトごとにデータが蓄積されるため、徐々にマッピング精度が向上する

## 5. 評価層：感性ワードへのスコア化

### 5.1 感性ワードの設計

#### ステップ1：感性ワードの選定

自社のブランド・製品で重視する感性価値を8〜12語程度選定する。以下は車室内UXの典型的な候補：

| カテゴリ | 感性ワード候補 | 対義語（参考） |
|---|---|---|
| **品質感** | 上質さ、精緻さ、堅牢感 | 安っぽさ、粗さ、華奢さ |
| **先進性** | 先進的、革新的、スマート | 旧式、保守的、野暮 |
| **安心感** | 安心、信頼、包容感 | 不安、懐疑、圧迫感 |
| **心地よさ** | 心地よい、自然、調和 | 不快、違和感、不協和 |
| **ブランド性** | （自社ブランド固有のワード） | — |

#### ステップ2：感性ワードとSD法形容詞対の対応表

各感性ワードを測定するためのSD法形容詞対を2〜3対ずつ設定する：

```
感性ワード「上質さ」
├── 洗練された ──── 粗野な
├── 丁寧な ──────── 雑な
└── 重厚な ──────── 軽薄な

感性ワード「先進的」
├── 新しい ──────── 古い
├── 知的な ──────── 原始的な
└── 洗練された ──── 野暮な

感性ワード「安心感」
├── 安心な ──────── 不安な
├── 穏やかな ────── 緊張する
└── 守られた ────── 無防備な

感性ワード「心地よさ」
├── 快適な ──────── 不快な
├── 自然な ──────── 不自然な
└── 調和した ────── ちぐはぐな
```

#### ステップ3：感性ワードと心理次元の対応仮説

| 感性ワード | 主対応心理次元 | 仮説パターン |
|---|---|---|
| **上質さ** | Valence（快）＋ Fluency（高） | 快感情が持続＋情報処理がスムーズ＝「引っかかりなく良い」 |
| **先進的** | Arousal（高）＋ Approach（接近） | 覚醒が高まり＋積極的に探索する＝「興味を引く新しさ」 |
| **安心感** | Valence（快）＋ Arousal（低〜中） | ポジティブかつ落ち着いている＝「安心」 |
| **心地よさ** | Valence（快）＋ Fluency（高）＋ Arousal（低〜中） | ポジティブ＋スムーズ＋落ち着き＝「心地よい」 |

### 5.2 感性スコアの算出方法

#### 方法1：重み付き加重平均（初期段階推奨）

```
感性スコア(上質さ) = w1 × Valence正規化値 + w2 × Fluency正規化値 + w3 × SD法スコア平均

  ※ 各値は0〜1に正規化
  ※ 重みは初期は均等(w1=w2=w3=0.33)、校正後に調整
```

#### 方法2：プロファイル一致度（中期以降推奨）

「理想的な感性パターン」を事前定義し、実測値との類似度を算出する：

```
                        理想プロファイル     実測プロファイル
Valence（快/不快）      0.8                  0.7
Arousal（覚醒度）       0.4                  0.5
Approach（接近/回避）   0.7                  0.6
Fluency（流暢性）       0.9                  0.8

感性スコア = 1 - ユークリッド距離(理想, 実測) / 最大距離
           = 1 - √((0.8-0.7)² + (0.4-0.5)² + (0.7-0.6)² + (0.9-0.8)²) / 2
           = 0.90（高い一致度）
```

## 6. 具体的計測プロトコル

### 6.1 事前準備

| 項目 | 内容 |
|---|---|
| 環境条件記録 | 室温、湿度、照明照度、騒音レベルを記録 |
| 機器キャリブレーション | アイトラッカー校正、EDA電極確認、骨格検知の原点合わせ |
| 被験者スクリーニング | 前日の睡眠6時間以上、カフェイン摂取2時間以上前、体調良好を確認 |
| 同意取得 | 生体データ取得・映像記録に関するインフォームドコンセント |
| HMI準備 | 評価対象のHMI設置 or スクリーン表示確認、切替手順確認 |

### 6.2 セッション進行表（1被験者あたり）

| 時刻(経過分) | 内容 | 実験者の行動 | 計測 |
|---|---|---|---|
| 0:00-0:05 | 説明・機器装着 | 実験説明、センサー装着 | 装着確認 |
| 0:05-0:08 | 安静ベースライン | 「楽にしてください」 | 生体計測開始 |
| 0:08-0:10 | 第一印象 | 「目を開けて自由に見てください」 | 視線＋生体＋映像 |
| 0:10-0:12 | 第一印象SD法 | 簡易SD法（5対）回答指示 | 主観記録 |
| 0:12-0:27 | 操作タスク | タスクカード提示（順次） | 全計測系フル |
| 0:27-0:32 | 自由探索 | 「自由に見て触ってください」 | 視線＋骨格＋生体 |
| 0:32-0:34 | 安静回復 | 「楽にしてください」 | 生体計測 |
| 0:34-0:38 | SD法（完全版） | 全形容詞対（12対）回答 | 主観記録 |
| 0:38-0:48 | 半構造化インタビュー | 感性ワードに沿った質問 | 音声記録 |
| 0:48-0:50 | 機器取外し・休憩 | — | — |
| | （2案目がある場合、5分休憩後に再度0:05から繰り返し） | | |

### 6.3 操作タスクの設計指針

機能UXと感性UXの両方を評価するため、タスクを2種類設計する：

#### タイプA：機能評価タスク（目標指向型）
具体的な操作目標を与え、達成過程を計測する。

| タスク例 | 計測する機能UX | 同時に計測される感性UX |
|---|---|---|
| 「エアコンを24℃に設定してください」 | 操作完了時間、エラー回数 | 操作中のEDA（ストレス/快適さ） |
| 「ナビで○○駅を目的地設定してください」 | タスク完了率、迷い時間 | 操作中のHRV（認知負荷→感性） |
| 「走行モードをスポーツに変更してください」 | 操作ステップ数 | モード変更後の視線パターン（変化の印象） |

#### タイプB：感性評価タスク（体験・探索型）
特定の操作目標を与えず、体験そのものの感性反応を観察する。

| タスク例 | 計測ポイント |
|---|---|
| 「メーターの表示を見て、印象を口に出してください」 | 発話内容＋視線パターン＋EDA |
| 「スイッチやノブを自由に操作してみてください」 | 接触順序・時間＋骨格＋EDA |
| 「室内を見回して、気になったところを教えてください」 | 視線走査パス＋発話＋前傾/後傾 |
| 「夜間イルミネーションを体験してください」 | EDA＋HRV＋表情＋SD法 |

### 6.4 インタビューガイド

半構造化インタビューの質問構造（感性ワードに沿って）：

```
1. 全体印象
   「この車室内空間の第一印象を一言で表現すると？」
   「最も印象に残った部分はどこですか？なぜですか？」

2. 感性ワード別深掘り
   [上質さ] 「この空間に『上質さ』を感じましたか？
             どの要素からそう感じましたか／感じませんでしたか？」
   [先進的] 「この空間に『先進的な印象』を持ちましたか？
             それはどこから来ていますか？」
   [安心感] 「この空間で『安心』や『落ち着き』を感じましたか？
             逆に『緊張』や『不安』を感じた瞬間はありましたか？」
   [心地よさ] 「操作していて『心地よい』と感じた瞬間はありましたか？
              逆に『違和感』や『不快』を感じた瞬間は？」

3. 比較（複数案の場合）
   「案Aと案B、どちらがより□□（感性ワード）ですか？」
   「その違いはどこから生まれていると思いますか？」

4. 改善提案
   「もっと□□（感性ワード）にするために、どうすれば良いと思いますか？」
```

## 7. データ分析手順

### 7.1 前処理

| データ種 | 前処理 |
|---|---|
| HRV | R-R間隔の外れ値除去（±3SD）、周波数分析（LF: 0.04-0.15Hz, HF: 0.15-0.4Hz） |
| EDA | トニック成分とフェイジック成分の分離、SCR（Skin Conductance Response）ピーク検出 |
| 視線 | フィクセーション検出（I-VT法、閾値30°/s）、AOI（Area of Interest）別集計 |
| 骨格 | ノイズ除去（移動平均）、前傾角度・リーチ距離の時系列化 |
| SD法 | 逆転項目の反転処理、因子分析 or 項目別集計 |

### 7.2 心理次元の算出

| 心理次元 | 算出方法 |
|---|---|
| Valence | (SD法「快-不快」系スコア平均 × 0.5) + (EDAトニックレベル変化の符号付き正規化 × 0.3) + (表情スコア × 0.2) |
| Arousal | EDAフェイジック頻度の正規化 × 0.4 + HRV-LF/HF比の正規化 × 0.3 + 呼吸数変化率 × 0.3 |
| Approach/Avoidance | 前傾角度正規化 × 0.3 + 視線滞留時間正規化 × 0.3 + 接触時間正規化 × 0.2 + 接触回数正規化 × 0.2 |
| Fluency | (1 - サッケード頻度正規化) × 0.4 + (1 - ためらい時間正規化) × 0.3 + (1 - 操作エラー率) × 0.3 |

**※ 重み係数は初期値。校正プロセス（4.3節参照）を経て調整する。**

### 7.3 感性スコアの算出

1. 各心理次元を0〜1に正規化（ベースライン区間を基準）
2. 感性ワードごとの重み付き加重平均を計算
3. 100点満点に換算
4. 設計意図プロファイルとの一致度を算出

### 7.4 統計的比較

| 比較の種別 | 統計手法 | 必要サンプルサイズ（参考） |
|---|---|---|
| 2案間比較 | 対応ありt検定 or Wilcoxon符号順位検定 | 12〜15名 |
| 3案間比較 | 反復測定ANOVA or Friedman検定 | 12〜20名 |
| 改善前後比較 | 対応ありt検定 + 効果量（Cohen's d） | 10〜15名 |
| 生体×主観の相関 | Pearson/Spearman相関 + 回帰分析 | 20名以上（校正用） |

## 8. 校正と成熟のプロセス

### 8.1 レベル1（初回〜3回目）：仮説検証期

- 主観評価（SD法＋インタビュー）を「正解データ」として使用
- 生体・行動データと主観データの相関を検証
- 相関の低い指標は除外 or 変換ロジックを修正
- **この段階での感性スコアは参考値**

### 8.2 レベル2（4回目〜10回目）：安定化期

- 蓄積データから回帰モデルを構築
- 個人差の影響を検討（被験者プロファイル×感性反応のクラスタリング）
- 製品カテゴリ別（ノブ系、画面系、照明系等）の感性パターンライブラリ構築開始
- **感性スコアの信頼区間を提示可能に**

### 8.3 レベル3（11回目以降）：予測モデル期

- 機械学習モデル（Random Forest, XGBoost等）による自動推定
- HMI仕様→感性スコアの予測精度が実用レベルに到達
- **将来のSynthetic User（感性AI）の訓練データ基盤が完成**

## 9. 実施上の注意事項

### 9.1 倫理的配慮
- 生体データは個人情報に該当する場合がある。社内倫理委員会 or 相当組織への確認
- 被験者には中止の自由を明示
- データの保存期間・アクセス権限の明確化

### 9.2 計測品質の確保
- EDA電極は評価開始5分以上前に装着（安定化のため）
- アイトラッカーは各被験者で校正実施
- 被験者の利き手・視力矯正状況を記録
- 空調の風がEDA電極に当たらないよう配慮

### 9.3 バイアスの制御
- タスク提示順のランダム化（ラテン方格法等）
- 実験者の声色・態度の統一（ダブルブラインドは困難だが、手順の標準化で対応）
- SD法の形容詞対提示順をランダム化
- 「正解がある」と被験者が感じないよう教示文を工夫
